Avec des langages traditionnels comme Python, C++ ou Java, construire une interface utilisateur convaincante peut vite devenir laborieux. Pourtant, combien de temps passez-vous sur Facebook, Twitter ou WhatsApp ?  
En 2025, tout fonctionne sur le Web, et ignorer ces technologies n'est tout simplement pas une option.

## Vocabulaire

Les technologies web reposent largement sur un modèle **client-serveur** :

- **Côté serveur** (pensez cloud) : le _backend_, qui gère la logique métier, le stockage de données et la gestion des requêtes.
- **Côté client** (pensez navigateur) : le _frontend_, qui se charge d'afficher et d'interagir avec le contenu pour l'utilisateur.

Par exemple, Facebook dispose d'une architecture backend énorme (centres de données, codes complexes, etc.) destinée à répondre aux requêtes des navigateurs modernes (Chrome, Firefox, Edge, Safari, …) et des applications mobiles. Ces interfaces, qui interagissent directement avec l'utilisateur, constituent le _frontend_.

## Périmètre du cours

Dans ce cours, nous nous concentrerons exclusivement sur le **frontend**.  
L'objectif est de vous donner les clés pour comprendre et exploiter les technologies de base supportées par tous les navigateurs :

- **HTML** : structure et contenu.
- **CSS** : mise en forme et style.
- **JavaScript** : comportement et interactivité.

Nous laisserons de côté, pour l’instant, les surcouches évolutives comme React, Angular ou d’autres frameworks, trop nombreux et changeants pour être abordés ici.

## Le trio fondamental

- **HTML : Le contenu**  
  Tout ce qui apparaît sur une page web est le résultat d’un fragment HTML.  
  Il définit la structure et la sémantique du contenu présenté à l’utilisateur.

- **CSS : Le style**  
  Le CSS détermine l’apparence visuelle de la page.  
  Grâce à lui, vous pouvez personnaliser la mise en forme et l'esthétique de votre contenu.

- **JavaScript : Le comportement**  
  JavaScript est un langage de programmation complet qui permet de rendre votre page interactive.  
  Il vous offre la possibilité de répondre aux actions de l’utilisateur et de gérer des opérations en temps réel.

## Ce que nous n'abordons pas

Dans le cadre de ce cours, nous n'aborderons pas :

- Les technologies **backend** (ex. : Apache, Nginx, Django, API REST, …)
- Les protocoles réseau sous-jacents (HTTP, HTTPS, WebSockets, etc.)

Ces sujets feront l'objet d'un cours complémentaire prévu plus tard dans l'année.

## Pourquoi les technologies web sont-elles si passionnantes ?

Les outils d'interface graphique traditionnels (comme .NET, Java Swing, Qt, …) reposent sur un modèle de widgets souvent rigide et limité.  
Le web, quant à lui, offre une interaction bien plus flexible :  
n'importe quelle partie de l'écran peut devenir interactive.

De plus, la séparation nette entre le contenu (HTML) et le style (CSS) permet une collaboration efficace entre designers et développeurs.  
Et cerise sur le gâteau, ces technologies sont **ultra-portables** : elles fonctionnent sur tout, des ordinateurs de bureau aux mobiles, sans nécessiter d'installation particulière.

## Au-delà du navigateur

Gardez à l'esprit que les technologies frontend ne se limitent pas aux navigateurs web.  
Elles peuvent servir à créer :

- Des applications **standalone** (fonctionnant en dehors du navigateur)
- Des applications mobiles natives (iOS ou Android)
- Des extensions ou outils pour des éditeurs de code comme VS Code

## JavaScript : Un outil polyvalent

Bien que JavaScript soit initialement connu pour dynamiser les interfaces web, il se révèle aujourd'hui être un outil de programmation généraliste.  
Grâce à des environnements comme **Node.js**, il est de plus en plus utilisé pour développer des applications côté serveur, confirmant son rôle central dans l'écosystème de développement moderne.

## Objectifs du cours

- **Découvrir** l'immense potentiel des technologies web.
- **Maîtriser** les concepts fondamentaux d'HTML, CSS et JavaScript.
- **Identifier** et comprendre les défis spécifiques liés aux interactions asynchrones et aux communications réseau.
- **Construire** une base solide pour appréhender les buzzwords et outils actuels du développement web.
